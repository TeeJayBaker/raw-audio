## TL;DR (project-relevant)

**The natively-suited scoring rule for raw waveform generation.** Audio waveforms *are* paths in `R^d` (mono `d=1`, stereo `d=2`) — exactly the data type the signature kernel is built for. The paper introduces the **signature kernel scoring rule**

```
φ(F, o) = E_{X,X' ~ F} [ K^Sig(X, X') ] − 2 · E_{X ~ F} [ K^Sig(X, o) ]
```

with `K^Sig(X, Y) = ⟨S(k_x), S(k_y)⟩_H` the inner product of two path signatures lifted into an RKHS, proven (Thm 1) **strictly proper on the space of bounded-variation paths once basepoint+time augmentations are applied** to remove signature invariances. This is **the same shape as AudioDEAR's energy distance** `‖x_1−y‖+‖x_2−y‖−‖x_1−x_2‖` — both are `2·attraction − repulsion` kernel scoring rules over m=2 samples — but the kernel is `K^Sig` instead of `−‖·‖`, which **matches *path-structural* moments (curvature, Lévy area, iterated integrals, all higher-order temporal correlations) rather than only first-moment marginal sample distance.**

Drop-in upgrade path in our wiki:

1. **AudioDEAR's loss** `L = ‖x_1−y‖+‖x_2−y‖−‖x_1−x_2‖` → replace `−‖·‖` with `K^Sig`. Same `m=2` estimator, same training loop, same one-step head. The energy-kernel knob in [methods.md](../wiki/concepts/methods.md)'s scoring-rule section becomes signature-kernel.
2. **IMM's design space** — IMM Lemma 1 frames Consistency Models as the `M=1`, energy-kernel, `(s=0, t=1)` corner. Signature kernel is a **kernel-axis** upgrade orthogonal to particle-count and time-pair axes; `M≥2` with `K^Sig` is the natural composition.
3. **Drifting's `V = −∇ δF/δq` template** (from 2603.09936) admits any energy `F` with `F=0 ⇔ q=p`. The signature-kernel MMD `MMD_{K^Sig}` qualifies (strict propriety = characteristic kernel = MMD zeros iff equal). **Sig-kernel-WGF** is a constructable new point in the Drifting / W-Flow family.

Cost is the catch: the signature kernel is computed by solving a **Goursat PDE**, giving `O(dl)` time and `O(l²)` memory once GPU threads ≥ dyadic-grid size (Salvi 2021 `sigkernel`). For 1 s @ 22 kHz mono, `l = 22050` — quadratic memory is the audio-side bottleneck. **Numerical stability** is the other catch — the paper itself reports Inf/NaN at scale and fixes via heuristic `1/√(IJ)` pre-scaling and exponential RBF static kernel only; **linear and polynomial static kernels explode** (Appx G, Fig G.4). No CPU benchmark, no GFLOPs, no audio.

The paper's actual demonstration domain (15-day weather paths on a 64×32 grid, geopotential at 500hPa, U-Net forecaster trained prequentially against the sig-kernel score) is incidental to us; it's the *construction* and the strict-propriety theorem we need.

## What's in this paper that we actually want

### The score, and why this shape

For an ensemble forecast distribution `F` over paths and an observed path `o`, the **signature kernel scoring rule** (Eq. 1) is

```
φ(F, o) = E_{X, X' ~ F} [ K^Sig(X, X') ]  −  2 · E_{X ~ F} [ K^Sig(X, o) ]
```

Standard kernel-score form (Gneiting & Raftery 2007 / Steinwart-Ziegel 2021 Thm 1.1): for any **characteristic** kernel on a compact path space, this score is strictly proper. The construction is identical in shape to Szekely's energy score; the work is in proving `K^Sig` is characteristic on path space.

The minimum-prequential variant for training a sliding-window generator (Pacchiardi et al. 2024 framework) is

```
φ̂_{k,l,m}  =  arg min_φ  Σ_t [ (1 / m(m−1)) Σ_{r≠s} K^Sig(X^r_p, X^s_p)  −  (2/m) Σ_r K^Sig(X^r_p, o_{t+1:t+l,p}) ]
```

with `m` ensemble samples per training step, `l` path length, `p` indexing patches in space (their geometry, irrelevant for audio). **For us, `m=2` is the most useful starting estimator** — exactly mirrors the AudioDEAR loss but with the signature kernel.

### The signature, in one paragraph

A `d`-dim path `X : [a,b] → R^d` is mapped to an **infinite sequence of iterated integrals**

```
S(X)^{i_1, ..., i_k}_{a,b}  =  ∫_{a<t_1<...<t_k<b}  dX^{i_1}_{t_1} ··· dX^{i_k}_{t_k}
```

over all index tuples `(i_1, ..., i_k) ∈ {1, ..., d}^k` and all orders `k`. Properties relevant to us:

- **Universal-nonlinearity:** linear functionals of the signature can `ε`-approximate any continuous function on bounded paths (Chen 1957 / Lyons; universal-nonlinearity property).
- Uniquely characterises a path up to translation and time reparametrisation (kills both with basepoint+time augmentations).
- Order-2 signature `S^{i,j} − S^{j,i}` = signed **Lévy area** between the `i`-th and `j`-th component path — captures curvature/coupling between dimensions. **For stereo audio: the Lévy area between L and R channels is a phase / spatialisation feature with no analogue in standard L2 losses.**

The signature is the path-analogue of moments: matching it matches *path-structural* statistics. Energy distance matches sample-marginal first moments only.

### Augmentations are mandatory (not optional)

The raw signature is invariant to translation and time-reparametrisation, so the bare signature kernel is **not** characteristic on paths. Two augmentations restore uniqueness:

- **Basepoint augmentation** `φ^b(x) = (0, x_1, ..., x_n)` — prepends a zero vector. Diagonal signature terms become `(X^i_b − 0)^k / k! = (X^i_b)^k / k!`, sensitising to translation.
- **Time augmentation** `φ_t(x) = ((t_1, x_1), ..., (t_n, x_n))` — adds an explicit time coordinate.

Without these the strict-propriety theorem fails; both are required and both are cheap. **Audio-relevant:** time augmentation makes the kernel insensitive to sample-rate jitter / resampling, which matters if waveforms are loaded at different rates during training.

Other augmentations exist (lead-lag for quadratic-variation sensitivity, log-signature for dimension reduction) — paper does not use them due to instability at high `d`; flagged as future work.

### The kernel trick and the Goursat PDE

The full signature is infinite-dim; the signature kernel `K^Sig(X, Y) = ⟨S(k_x), S(k_y)⟩_H` collapses it to a single scalar without instantiating any signature terms. **Salvi et al. 2021** prove it solves a Goursat (boundary-value) PDE with time complexity:

- `O(d · l²)` worst case for paths of length `l` in dimension `d`.
- `O(d · l)` once GPU thread count ≥ dyadic-grid size (paper uses dyadic order 1; bigger orders trade speed for accuracy).

The implementation used is the `sigkernel` Python package. Memory is `O(l²)` (the Goursat grid).

**For audio (`d=1` or `d=2`, `l ≈ 10^4 – 10^6`):**
- 0.1 s @ 22 kHz → `l = 2205`, `l² ≈ 5 · 10^6` — fits trivially.
- 1 s @ 22 kHz → `l = 22050`, `l² ≈ 5 · 10^8` — borderline (~2 GB float32 per pair).
- 10 s @ 22 kHz → `l = 220500`, `l² ≈ 5 · 10^{10}` — does not fit on a 4090.

**The natural audio operating regime is short samples (the project's stated 0.1–1 s sweet spot) or strided / dyadic-windowed paths.** Patching/striding the waveform before signature computation is the obvious adaptation; paper does the spatial analogue on weather grids (24 16×16 patches with overlap) and finds it preferable to global computation.

### Strict propriety theorem (the load-bearing claim)

**Theorem 1 (paper).** Under (i) a characteristic static kernel (RBF qualifies); (ii) paths of bounded variation on a compact domain; (iii) uniqueness via basepoint+time augmentation — `φ(F, o)` is strictly proper.

Proof (Appx E) is a direct invocation of Steinwart-Ziegel 2021 Thm 1.1: kernel score is strictly proper iff `K^Sig` is characteristic on the path space. Compactness via Arzelà–Ascoli (uniform boundedness + equicontinuity + closedness of piecewise-linear paths with knots at fixed grid). Characteristic-ness via Issa et al. 2023 Props 3.1/3.3 (signature kernel induced by a characteristic static kernel is characteristic on bounded-variation paths once augmentations remove invariances).

**For us, what this guarantees:** the loss landscape has a *unique global minimum* at the true path distribution. SGD on the empirical signature kernel score converges (under stationarity/mixing) to a model that matches `Q` exactly — no spurious minima, no mode collapse from the loss itself. **The same propriety property that motivates IMM's Lemma 1 framing of consistency models, but on path data rather than marginal sample data.**

### Discriminative power (Appx G is the most-relevant ablation for us)

Toy 2-D vegetation/fire-spread simulator; sweep one parameter; compare scores as discriminators.

- On a **temporally-correlated** parameter (vegetation growth): sig-kernel, energy, MSE, variogram all curve similarly. **Sig kernel doesn't beat the others here** — temporal-only correlation is what marginal-moment scores already capture.
- On a **spatio-temporally correlated** parameter (fire spread): **sig-kernel score is sharper than all three competitors** (Fig G.3). Asymmetric curve (drops harder when spread intensity *increases* because fire propagation creates more spatially-correlated paths) — the kernel reads the joint structure that marginal scores miss.
- Static kernel choice (Fig G.4): **RBF bounded in `(0, 1]` — only stable choice.** Linear and polynomial (`d=2`) explode (linear hits ~80, polynomial ~`10^9`) and are numerically unusable in training. This is hard-locked: **use RBF as the static kernel, σ=1, dyadic order 1.**

### Numerical instability (§4.3 — the real-world cost)

The paper itself reports Inf/NaN values and scores collapsing to 1.0 at moderate path length. Two compounding factors:

- Static kernel choice (RBF is exponential-decay-bounded; linear/polynomial unbounded).
- Goursat PDE solver accumulates error when path differences are large.

Their fix: pre-scale paths to roughly unit range (standard normalise using mean/std of observation), then multiply by `1/√(I·J)` (the longitude × latitude dimensionality) — a heuristic with no theory but solved their problem up to `l=2048` evaluation / `l=256` training. Morrill et al. 2021 only validated augmentations up to `d=8`; this paper pushes to `d=2048` and reports instability remains for linear/polynomial kernels.

**For audio:** the analogue of `1/√(I·J)` is `1/√d` where `d` is the channel count (1 or 2) — trivial. The path-length scaling is what bites: longer audio paths increase the Goursat solver error budget, and the paper has not characterised this regime past `l=2048`.

### Training results (incidental to us, but for completeness)

- WeatherBench 2 ERA5 reanalysis, geopotential 500hPa, 64×32 resolution, 2010–2018 split.
- Generative U-Net (depths 32/64/128/256 — small for this domain, ~few M params), 10-step observation context, latent Gaussian `z`, sliding-window prequential generation.
- Path lengths trained: `l ∈ {2, 5, 10, 15}`. 24 patches of `16×16` with 3-lat × 8-lon overlap, dimension 256 each.
- 3 ensemble members per training step (paper notes `m ∈ {3, …, 50}` gave equivalent metrics, so the cheap end was kept).
- LR `∈ {1e-2, …, 1e-6}` × `l ∈ {2,5,10,15}` swept, 100-epoch budget with early stopping.

Results (Table 1): NCRPS < 1 across all path lengths (beats climatology), calibration error plateaus at 0.2 from `l=5` upward, `R²` drops to ~0 at `l=15` (model matches climatology at long horizons). **No CPU number, no FLOPs, no inference-time benchmark.**

## How this translates to raw audio

### The minimum-viable port

Audio waveforms are paths. The path-space view doesn't need any reframing — `X^i_t` is sample `i` of channel `t`, time augmentation puts the sample index into the path. The signature kernel is dimension-agnostic; static-kernel hyperparameters (RBF σ=1) are dimension-agnostic up to a pre-scaling step that matches the audio's amplitude range (waveforms are already in `[-1, 1]` — no rescaling needed if RBF σ is chosen accordingly).

**The drop-in upgrade to AudioDEAR's loss:**

Current AudioDEAR (Szekely energy distance, m=2):
```python
L = ‖x_1 − y‖ + ‖x_2 − y‖ − ‖x_1 − x_2‖
```

Sig-kernel scoring rule (m=2 estimator):
```python
L = K^Sig(x_1, x_1) + K^Sig(x_2, x_2)
    − 2·K^Sig(x_1, y) − 2·K^Sig(x_2, y)
    + 2·K^Sig(x_1, x_2)
    + (self-similarity term K^Sig(y,y) drops if we use the score form)
```

The actual training surrogate (Eq. 1 minus an `o`-only term that drops as constant w.r.t. generator parameters):
```python
L = (1/m(m−1)) Σ_{r≠s} K^Sig(x_r, x_s)  −  (2/m) Σ_r K^Sig(x_r, o)
```

with `m=2` this collapses to one cross-pair attraction + one repulsion term. Same shape, kernel is `K^Sig` instead of `−‖·‖`.

### Audio dimensionality budget

Path-length scaling is the real cost:

| Sample | `l` (22 kHz mono) | `l²` (Goursat grid) | Goursat memory (f32) | Fits 4090? |
|---|---|---|---|---|
| 0.1 s | 2 205 | 4.86 · 10^6 | 19 MB | yes (trivial) |
| 0.5 s | 11 025 | 1.22 · 10^8 | 486 MB | yes |
| 1.0 s | 22 050 | 4.86 · 10^8 | 1.95 GB | borderline (BS=1 only) |
| 5.0 s | 110 250 | 1.22 · 10^{10} | 49 GB | **no** |
| 10.0 s | 220 500 | 4.86 · 10^{10} | 195 GB | **no** |

**Patching the path is the obvious fix** and what the paper does on weather grids: split the waveform into `P` overlapping chunks of length `l_patch`, compute sig kernel per patch, sum the scores. This costs `P · l_patch²` instead of `(P·l_patch)²`. For 1 s @ 22 kHz with `l_patch = 1024`, `P ≈ 22`, total cost `22 · 10^6` — trivial.

**Patching destroys the cross-patch path structure**, which is the whole point of the signature. The right knob is *overlapping* patches with non-trivial overlap (paper uses ~3-lat / 8-lon overlap on 16×16 patches; the audio analogue is half-window overlap). Long-range temporal structure beyond patch length is lost — this is the price.

**Striding / decimation** is the other obvious adaptation: instead of `l = 22050`, decimate by `s` to `l/s` and treat the decimated path as the signature input. Loses high-frequency information but preserves global structure. Multi-resolution sig-kernel (sum of scores across decimation levels) is the natural extension; paper does not explore it.

### Stereo: the Lévy area between L and R is a free phase feature

For mono `d=1`, only diagonal signature terms survive — no cross-channel structure. **For stereo `d=2`, off-diagonal Lévy-area terms `S^{L,R} − S^{R,L}` quantify the signed area between L and R waveforms**, which is a coupling / spatialisation feature with no clean L2-loss analogue. The signature kernel gives this for free; an energy-distance loss does not.

### Where this composes (existing wiki)

1. **AudioDEAR loss (energy distance) → sig-kernel score.** Same training loop, same `m=2` estimator, same one-step head. Strictly more expressive (matches path-structural moments, not just sample moments). **This is the cleanest first experiment.**
2. **IMM kernel axis.** IMM frames CM/GMMN/IMM as points in `(time-pair, M, kernel)` space. Signature kernel is a fourth kernel option alongside energy, Laplace, pseudo-Huber CPD. At `M=2` it should match path-structural moments; at `M≥4` it should match higher-particle path-distributional moments. Lemma 1 of IMM identifies the first-moment energy-kernel corner as the worst — sig kernel is a non-trivial upgrade because it adds infinitely many path-structural moments at once.
3. **Drifting / W-Flow `V = −∇ δF/δq` template.** Sig-kernel MMD `MMD_{K^Sig}` qualifies as an `F` (strictly proper ⇒ characteristic ⇒ `MMD=0 ⇔ q=p`). The drift `V = ∇ δMMD/δq` is computable in closed form via the Goursat-PDE-evaluated kernel gradient (autodiff through `sigkernel`). **Sig-kernel-WGF** is a constructable new point in the Drifting family — distinct from MMD-WGF on a non-signature kernel (which W-Flow Tab 2a shows is worse than Sinkhorn-WGF on images), because the signature is the *audio-relevant* generalisation of MMD's matched moments.
4. **Pacchiardi prequential training.** The paper's training framework (one-step → sliding-window prequential SR minimisation, Pacchiardi et al. 2024) is designed for sequence-conditional generation. **For unconditional audio gen we don't need the sliding window** — single-step scoring rule minimisation directly on (forecast distribution, observed audio) pairs. The framework simplifies, not the loss.

### Where this does *not* obviously help

- **Multi-step diffusion / FM training** doesn't benefit directly — those train against a noise-prediction target, not a distribution-matching score. The sig-kernel score is the right *evaluation* of such models on path data (a sig-kernel FAD analogue), but not the right training loss.
- **Vocoders** (HiFi-GAN, RFWave, Flow2GAN, PeriodWave-Turbo) are conditioned generators — the conditional distribution `p(x | mel)` is pinned tightly enough by the condition that scoring-rule training is overkill; mel-L1 + GAN works. Sig-kernel might help on the *conditional-diversity* axis but the baseline is already good.
- **Long audio** (≥ 5 s) hits the `l²` Goursat memory wall. Patching loses the global structure that motivates the signature in the first place. **The audio sweet spot is 0.1–1 s** — fortunately exactly the project's stated target.

## Strengths

- **Strict propriety on path space** — a unique-global-minimum guarantee on the loss landscape that no other audio-domain training objective offers. Energy distance is strictly proper on sample distributions; sig-kernel is strictly proper on *path* distributions, which is the right object for audio.
- **Captures path-structural moments** — curvature, Lévy area, iterated integrals up to infinite order, all in one kernel evaluation. The natural multi-resolution structural loss for waveforms.
- **Drop-in upgrade to AudioDEAR.** Same loss shape, same training loop, same `m=2` estimator.
- **Free stereo phase / spatialisation feature** via off-diagonal Lévy area on `d=2` paths.
- **Goursat-PDE solver scales linearly in `l`** once GPU thread count is sufficient (Salvi 2021).
- **`sigkernel` Python package exists** — no implementation cost.

## Weaknesses / open questions

- **`O(l²)` Goursat memory** caps the natural audio length at ~1 s @ 22 kHz on a 4090. Patching is necessary but kills the cross-patch path structure; the right patching/striding schedule for waveforms is unstudied.
- **Numerical stability** — paper itself reports Inf/NaN and a heuristic `1/√(I·J)` fix that has no theoretical grounding and was validated only up to `l=2048` evaluation / `l=256` training. **At audio `l ≈ 10^4` the stability margin is untested.**
- **Static kernel choice is hard-locked to RBF.** Linear and polynomial explode (Appx G Fig G.4). No exploration of perceptually-relevant audio static kernels (Mel-distance, AudioMAE-distance) as the static kernel inside the signature — possibly a good idea, totally untested.
- **σ=1 is heuristic** — the paper picks RBF σ=1 from Issa et al. 2023 with no audio justification. Audio bandwidth choice for the static kernel is an open hyperparameter.
- **No CPU benchmark anywhere.** GPU-only evaluation, no FLOPs. The sig-kernel score is for *training* — inference cost is determined by the generator (1 NFE for a one-step head), so the *inference* benchmark is the standard generator-architecture benchmark — but training-time cost on CPU (relevant for ablations / small-data experiments) is unmeasured.
- **No comparison to energy distance on the same problem.** The Appx-G discriminative-power figure compares to MSE / variogram / energy, but only on a 2-D simulator. **The image-domain or audio-domain head-to-head sig-kernel-loss vs energy-distance-loss at matched generator architecture is the missing experiment.**
- **Sparsity** — Goursat solver assumes dense signatures. Audio waveforms have silent regions (zero-inflated) that produce sparse signatures; paper flags this as an open problem (precipitation has the same issue on the weather side).

## Things to test in this repo

- **Sig-kernel loss vs AudioDEAR's energy distance on matched 1-step head.** Same backbone, same training data (drums, 0.5–1 s), same `m=2` estimator. Compare FAD / Vendi / Coverage at fixed compute. **This is the single most informative experiment** — answers whether the kernel upgrade actually helps on raw audio.
- **Path-length × static-kernel sweep for numerical stability on waveforms.** Push `l` from 256 to 22050, swap RBF for any perceptual static kernel (Mel-L1, AudioMAE-distance as the static kernel inside the signature); identify the stability frontier.
- **Patching strategy for 1 s audio.** Non-overlapping vs 50%-overlap vs dyadic windows vs strided/decimated. Trade off `P · l_patch²` cost against retained cross-patch structure.
- **Stereo Lévy area as a free spatialisation feature.** Train on stereo drums (kicks panned hard, hi-hats panned soft) and check whether the sig-kernel head reproduces L/R imbalance correctly without an explicit panning condition. Energy-distance baseline cannot.
- **`m=2` vs `m=4` ensemble size in the prequential estimator.** Paper sees `m=3` matches `m=50` on weather metrics — does this hold for audio, where higher-frequency structure may want more particles to estimate?
- **Sig-kernel as evaluation metric (audio-FAD analogue).** Use `K^Sig(F_gen, F_real)` distance on a held-out test set as an FAD replacement on path data. Direct port of the paper's evaluation use, plug into [evaluation.md](../wiki/concepts/evaluation.md).
- **Sig-kernel-WGF as a Drifting-family entry.** Construct `V = ∇ δMMD_{K^Sig}/δq` via autodiff through `sigkernel`, drop into the Drifting / W-Flow training loop. **Compare against MMD-WGF (W-Flow Tab 2a baseline)** — does the signature kernel beat a plain kernel here? If yes, the Drifting family gets a new top entry on audio.
