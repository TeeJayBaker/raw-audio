## How this is updated

The `ingest-paper` skill writes a per-paper MD at `papers/<slug>.md` and appends or refreshes a row in the table below. The table is kept in **most-recently-ingested first** order.

The TL;DR column is **project-relevant** (raw sample-space audio gen on CPU), capped at ~40 words, no embedded tables, no number dumps, no contrastive comparisons against other rows — those live in the per-paper MD. See the `ingest-paper` skill for the full contract.

If a paper's reading changes substantially, update both `papers/<slug>.md` and its row here — but tighten the row, don't grow it.
