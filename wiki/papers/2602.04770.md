# TL;DR — why this matters for raw-audio CPU gen

A new "from-scratch" recipe for **1-NFE generators** that does not pass through a multi-step teacher at all. The trick: rather than integrating a learned vector field at inference time (diffusion / flow / consistency), Drifting Models put the iteration *inside training* by defining an explicit, anti-symmetric **drifting field** `V_{p,q}(x)` that pulls generated samples toward the data distribution and pushes them away from each other. Training minimises `||V||²` on generated samples — at equilibrium `V ≡ 0` ⇔ `q = p`. Inference is a single forward pass of a vanilla DiT. Sets a new SOTA for 1-NFE pixel-space ImageNet 256² (FID 1.61) and latent-space (FID 1.54), beating GANs and previous from-scratch one-step diffusion variants like MeanFlow / Shortcut / EPG. For us: a candidate **one-step, GAN-free, teacher-free** path to a CPU-cheap raw-audio generator. Two important caveats: (a) the loss needs a strong pretrained feature encoder φ for high-d data — the authors could not get pixel-ImageNet to work *without* one — so audio transfer hinges on having something like AudioMAE/CLAP/EnCodec; (b) it depends on a contrastive **positive batch** (real samples in the loop), so it is more like a discriminator-free GAN than like distillation.

# What problem does it solve

The CPU-inference target wants **1-NFE generation**. The existing 1-NFE families each have a tax:

- Distillation (DMD, ICT, Shortcut, MeanFlow as distillation, EPG): needs a working multi-step teacher first → double the training cost, and you inherit the teacher's bias.
- GANs (StyleGAN-XL, GigaGAN): adversarial instability, mode collapse, no explicit data-likelihood-style objective.
- Consistency models trained from scratch (iCT, sCM): unstable, sensitive to schedule, lag SOTA.

Drifting Models propose a single non-adversarial, non-distillation objective whose minimiser at equilibrium provably has `V=0` ⇒ (under mild assumptions) `q=p`. The iteration that diffusion does at inference time is folded into the SGD trajectory itself — `q_i = f_{i#}p_ε` evolves with training step `i` rather than with denoising step `t`.

# Method, the part that distinguishes it from neighbours

## Pushforward at training time, not inference time

Define `q = f_{#}p_ε`, the distribution of the network's output under noise input ε. Goal: `q ≈ p_data`. Across training iterations, `q_i = f_{i#}p_ε` forms a sequence of distributions. The update `Δx_i = x_{i+1} − x_i` (induced by an SGD update to f) is the **drift** of sample x.

## The drifting field V_{p,q}(x)

A vector field with the key property `V = 0 ⇔ q = p`. Imposed by **anti-symmetry**: `V_{p,q}(x) = −V_{q,p}(x)` (Prop 3.1) — swapping the roles of data and model just flips the sign, so when `p = q` the field must vanish identically.

Their concrete instantiation is **mean-shift-style**, with attraction toward data and repulsion from the current model distribution:

```
V_{p}^{+}(x) = (1/Z_p) E_{y⁺~p}[ k(x, y⁺) (y⁺ − x) ]
V_{q}^{-}(x) = (1/Z_q) E_{y⁻~q}[ k(x, y⁻) (y⁻ − x) ]
V_{p,q}(x)  = V_p^{+}(x) − V_q^{-}(x)            (Eq. 10)
```

with kernel `k(x, y) = exp(−(1/τ) ||x − y||)` (ℓ₂, not squared; softmax-normalised over the y-axis and over the x-axis within a batch — the latter is unusual and ablated).

## Training objective — a fixed-point regression

At equilibrium the network must satisfy `f(ε) = f(ε) + V_{p,q}(f(ε))`. They convert this into a loss with the iteration *built in* via stop-gradient:

```
L = E_ε [ || f_θ(ε)  −  stopgrad( f_θ(ε) + V_{p,q_θ}(f_θ(ε)) ) ||² ]
        prediction          frozen target = drifted version
```

(Eq. 6.) Equivalently, the gradient of L is `−2 V`, so this is *V-centric gradient descent on the generator outputs*. Implementation is 7 lines (Alg. 1): generate a batch `x = f(ε)`, draw real `y⁺ ~ p_data`, use the **same batch** `x` as negatives `y⁻ ~ q`, compute V, regress x toward `stopgrad(x + V)` with MSE.

Crucially **no backprop through V**: V depends on the distribution `q_θ`, which would be infeasible to differentiate.

## Feature-space variant (this is where it actually works at high d)

Plain pixel-space drift collapses on ImageNet. The fix is to compute the drift in a **pretrained feature space** φ:

```
L = E [ || φ(x) − stopgrad( φ(x) + V_φ(φ(x)) ) ||² ]
```

Extended to **multi-scale, multi-location** features from a ResNet-style encoder (Eq. 14). The encoder is a *custom* latent-MAE (ResNet trained with MAE objective on the same VAE latent space the generator outputs); they also try MoCo, SimCLR, standard MAE, ConvNeXt-V2, and combinations. The encoder is **frozen** during generator training. Gradients backprop through the encoder *but only through B-sized x*; positives/negatives are detached via stop-grad on the target.

This relegates the method to "needs a good φ" — every other choice (estimator, kernel, normalisation) is robust, but **choice of φ matters most** (Tab 3), and the authors explicitly state they could not make pixel-ImageNet work without one.

## Stochastic estimation, normalisation, multi-temperature

- Each step: `N_pos` real positives + `N_neg` generated negatives. Negatives are the same generator outputs the loss is regressing — so the contrastive "negative bank" is free (Alg. 1 line `y_neg = x`).
- Feature normalisation `S_j` makes the kernel scale-free across features of different magnitudes; drift normalisation `λ_j` does the same for V.
- Multiple kernel temperatures τ ∈ {0.02, 0.05, 0.2} are summed — corresponds to mean-shift at multiple bandwidths simultaneously. Robust to choice of τ.
- The expected loss value `E[||V(f(ε))||²]` is itself a useful training-progress signal — the loss is a real distance proxy, not just a surrogate gradient.

# Architecture

Generator is **DiT/DiT/2** for latent and **DiT/16** for pixel — straight transformer, no special structure. Input ε is patchified Gaussian noise (32×32×4 latent → 256 tokens, or 256×256×3 pixel → 256 tokens with 16×16 patch). Modern transformer tricks: SwiGLU, RoPE, RMSNorm, QK-Norm. AdaLN-zero for class + CFG conditioning. 16 in-context "register" tokens for class info. 32 **random style tokens** (StyleGAN-style codebook) added to conditioning — gives the network a discrete latent component on top of Gaussian noise (8.86 → 8.46 FID).

Sizes: B/2 ≈ 133M params, L/2 ≈ 463M params. Feature encoder is ~50M ResNet-MAE.

**Single-pass, non-iterative** at inference. CFG is *training-time* (sample CFG strength α per step from a distribution, condition the network on α), so no second forward pass at inference — important for our CPU goals.

# Training objective recap

```
L = Σ_j E [ || φ̃_j(x) − stopgrad( φ̃_j(x) + Ṽ_j(φ̃_j(x)) ) ||² ]
```

with normalised features φ̃, normalised drift Ṽ, summed over multiple feature scales/locations j and multiple temperatures τ. That is essentially the whole training story.

# Inference cost

**1 NFE.** A single DiT forward pass on a 256-token sequence. No solver, no second model. CFG is folded into training so the inference graph is just `f_θ(ε, c, α) → x`.

Pixel-space DriftingModel-B/16 reports 87 GFLOPs per 256² image — for reference, StyleGAN-XL reports 1574 GFLOPs to reach a worse FID 2.30. That gap is the whole point. We don't yet have CPU latency from the paper; we'd need to measure DiT/2-B on CPU at our sequence lengths to know if this hits our budget.

# Headline results (1-NFE, no distillation)

| Setting | Model | FID | Compare-to |
|---|---|---|---|
| ImageNet 256², latent | Drifting L/2 (463M+49M) | **1.54** | iMF-XL/2 1.72, MeanFlow-XL/2 3.43, AdvFlow-XL/2 2.38, Shortcut-XL/2 10.60. Beats all 1-step; competitive even vs DiT-XL/2 250-step at 2.27. |
| ImageNet 256², pixel | Drifting L/16 (464M) | **1.61** | StyleGAN-XL 2.30 (1 NFE), EPG-L/16 8.82 (1 NFE), JiT-G/16 1.82 (200 NFE). |
| ImageNet 256², latent (B-size) | Drifting B/2 (133M+49M) | 1.75 | beats prior XL-size 1-step entries. |
| Diffusion Policy (robotics control) | Drift 1-NFE replacement | matches or beats 100-NFE DP | Across 6 tasks (Lift, Can, ToolHang, PushT, BlockPush, Kitchen). |

Ablations that translate:

- Anti-symmetry is essential. Removing it (e.g. 1.5× attraction only, repulsion only, V⁺ only) tanks FID from 8.46 to ≥41 or ∞ (177 for attraction-only). Equilibrium structure is load-bearing.
- Feature space is essential. Pixel-ImageNet with the same recipe **doesn't converge** without φ; pixel-space results only after the authors built a strong latent-MAE-then-decode + ConvNeXt-V2 stack.
- N_pos, N_neg help — typical contrastive-method scaling. Default 64/64 is fine but 128/128 is better.
- Multi-temperature {0.02, 0.05, 0.2} > single τ.
- Kernel x-axis softmax (their addition) > y-axis only > no normalisation (10.54 FID even without normalisation, so robust).
- Style embeddings help by ~0.4 FID.

# How this translates to raw audio

Three independent issues to think about.

## 1. Does the method's central idea transfer to 1D audio?

The drifting-field formulation is dimension-agnostic. It needs:

- A generator `f` mapping noise to data — fine: any 1D U-Net or causal-transformer generator works.
- A way to draw positives `y⁺ ~ p_data` in batch — fine: real audio clips from our corpus.
- A kernel `k(x, y)` measuring similarity in **some space** — this is where audio specifics enter.

The contrastive mean-shift structure (attract by positives, repel by negatives) is exactly what one would expect to work for audio too. The math doesn't care.

## 2. What's φ for audio?

**This is the crux.** The paper's central practical finding is that direct pixel-space training fails and the loss only converges when computed in a strong learned feature space. For raw audio this means we need a frozen φ that:

- Embeds an audio clip into a fixed-d vector (or, better, multi-scale feature maps for the multi-feature variant);
- Has well-shaped local geometry — kernel similarity `exp(−||·||/τ)` should correspond to perceptual similarity;
- Is *invariant in roughly the right ways* — we want clips that sound the same to be close, clips that sound different to be far.

Candidates (in rough order of likely fit):

- **AudioMAE / a custom in-house "waveform-MAE"** — direct analogue of what the paper uses. ResNet1D trained with masked-prediction on our own audio corpus. The paper makes a strong case (Tab 3) that a *custom* MAE-trained encoder is the right answer; off-the-shelf SSL encoders (SimCLR, MoCo) underperform.
- **EnCodec encoder embeddings** — already trained to reconstruct raw waveform, so phase information lives in there.
- **CLAP / MERT / Wav2Vec2** — semantic audio encoders, fine for high-level similarity but possibly *too* invariant for sample-level matching.
- **Mel-spectrogram + small CNN** — cheap baseline; phase-blind, so will likely repeat the pixel-space failure mode where phase-style artefacts are unpenalised.

The paper's multi-scale ResNet recipe (Tab 8: blocks [3, 4, 6, 3], widths {C, 2C, 4C, 8C}, base C=640, multi-scale features at 4 resolutions) is a strong default. The 1D analogue: 4-stage ResNet1D over a long waveform window, features pooled at 4 temporal resolutions.

## 3. Sample length and CPU-inference scaling

Our target is 0.1–10 s clips at music sample rates. Key numbers:

- DiT/2 with 256 tokens at 768/1024 hidden dim is light. A 1D analogue at e.g. 22.05 kHz, 1 s = 22 050 samples — with a patch size of 256 → 86 tokens, well below 256. At 44.1 kHz, 1 s = 44 100 → 172 tokens at the same patch. So sequence length is favourable.
- Inference is one transformer forward. CPU cost is dominated by the attention + MLP at 86–256 tokens × 768-d. Should be cheap; needs measurement.
- The catch: the **feature-space loss requires the encoder + (for pixel-space variant) the VAE decoder at training time**. Pixel-space training runs feature extractor on `f(ε)` output. For audio, we'd run φ on generated waveform every step — adds GPU memory, but doesn't touch inference latency.

## 4. Phase, stereo, perceptual issues

The pixel-space variant of the paper uses a *pixel*-MAE φ. The analogue here is: φ must operate on raw waveform (not on a phase-discarding spectrogram) for the kernel similarity to penalise phase artefacts. EnCodec or a waveform-MAE is the right shape. Mel-only φ is the obvious failure mode to avoid.

Stereo: drifting field is on samples in ℝᴰ, where D = waveform length × channels. Nothing in the method blocks stereo — just doubles D. Whether a stereo-trained φ ranks stereo-coherence well is an open empirical question.

## 5. Where this lands relative to FD-loss (2604.28190)

These are **two different one-step recipes** worth comparing directly:

| | FD-loss (2604.28190) | Drifting (this paper) |
|---|---|---|
| Required state | Frozen φ + **precomputed (μ_r, Σ_r)** offline | Frozen φ + **live batch of real positives** + live batch of negatives (generated) |
| Loss structure | Fréchet distance in φ-space (Gaussian moment match) | Pointwise mean-shift drift in φ-space (kernel-density contrastive) |
| Training path | **Post-trains** an existing pretrained generator (multi-step → 1-step, or 1-step base → better) | **Trains from scratch** as a 1-step generator |
| Trains from scratch? | Not their main use case — they post-train | Yes — primary use case |
| Needs a teacher? | No | No |
| Needs an adversary? | No | No |
| GPU footprint per step | Pretty light: φ + d×d matmul | φ + N_pos + N_neg samples in feature space |
| Failure mode | Saturation of φ (Gaussian assumption breaks); FID-only views mislead | Pixel-space fails outright; depends on φ being strong |

These could potentially be **stacked**: train from scratch with Drifting to a decent 1-step base, then post-train with FD-loss to further sharpen distributional match. Both keep inference at 1 NFE.

# Strengths

- **From-scratch 1-NFE without GAN, distillation, or consistency-style schedules.** The first published recipe I've seen that hits SOTA in this regime.
- **One simple loss, 7 lines of code.** Conceptually clean: regress the model output toward its own drifted target with stop-grad.
- **Anti-symmetry is a sharper equilibrium structure** than just minimising a discrepancy like MMD — it pins the field, not the scalar.
- Robust to a wide range of hyperparameters: kernel normalisation, temperature, queue size, etc. Most "tricks" can be left at default.
- CFG without doubling NFE: CFG strength is a training-time conditioning input. Direct benefit for our CPU-inference budget — many diffusion CFG implementations double inference cost.

# Limitations / things to test

- **φ-dependence is severe.** Pixel-ImageNet *doesn't work* without their custom latent-MAE+ConvNeXt-V2 stack. Audio likely repeats this: we'll need to invest in an audio φ before this method pays off.
- **No theory that V = 0 implies q = p in full generality.** The converse of Prop 3.1 is false; they argue (Appendix C.1) that their specific kernelised V *does* identify q=p under linear-independence assumptions, but it's an empirical claim in practice.
- Connection to MMD is intimate but not subsumed: their `V = V⁺ − V⁻` is equivalent to a particular MMD gradient surgery with normalised kernels (Appendix C.2). They couldn't get MMD-loss to work directly. So while drift is "MMD-with-anti-symmetry-and-V-centric-updates", the practical recipe is sharper.
- Negatives are reused from the same batch (`y_neg = x`). Fine on ImageNet, possibly thin for highly multi-modal audio corpora.
- No reported CPU latency anywhere in the paper. Their headline is GFLOPs, not wall-clock; for us, the **CPU wall-clock at audio sequence lengths is the headline number we need to measure**.

# Open questions for this project

1. **Cheapest audio φ that still works.** Concrete experiment: train a small ResNet1D-MAE on a corpus of 1 s waveform crops at 22.05 kHz; use it as φ for a 1D-DiT generator trained with the drifting loss. Compare CPU inference latency vs perceptual quality (FAD, listening tests).
2. **Does the V=0 equilibrium hold in 1D audio without retraining a custom φ?** The paper failed on raw pixels with off-the-shelf SSL encoders; will we fail on raw waveform with off-the-shelf VGGish/CLAP?
3. **Stereo:** does `||·||₂` in stereo waveform space induce a useful kernel, or do we need per-channel φ and a joint similarity?
4. **Length conditioning.** The paper's α (CFG strength) is a continuous training-time conditioning; we want clip-length conditioning at inference. Same mechanism should plug in.
5. **Compose with FD-loss.** Train from scratch with Drifting → post-train with FD-loss. Both are 1-NFE-preserving, both work in φ-space.
6. **MeanFlow vs Drifting head-to-head on raw audio.** Both are from-scratch 1-NFE; MeanFlow has a more diffusion-flavoured update, Drifting is contrastive. Worth a direct comparison.
