# TL;DR — why this matters for raw-audio CPU gen

IntMeanFlow is **the third ByteDance audio-MeanFlow paper from the same lab as [SplitMeanFlow](2507.16884.md)** (overlapping authors: Wei Wang, Rong Cao, Yi Guo, Zhengyang Chen, Kuan Chen) and constitutes a **structurally different JVP-avoidance recipe** from any other entry in the MeanFlow lineage. The recipe is **teacher-numerical-integration distillation**: an existing FM teacher does an `n`-step Euler integration over `[t, r]`; the displacement divided by `r − t` is the regression target for the student's average velocity `u_θ(z_t, t, r)`. **No JVP, no algebraic identity on the student, no self-bootstrap, no curriculum, no Taylor finite difference.** It is a pure mean-of-teacher-trajectory distillation — closer in spirit to multi-step trajectory distillation than to the from-scratch MeanFlow family, even though the parameterisation (`u_θ(z_t, t, r)` predicting average velocity) is identical. **The crucial distinction from SplitMeanFlow**: SplitMeanFlow's algebraic identity is computed *from three forward passes of the student itself* and never invokes the teacher's velocity; IntMeanFlow's "integral form" *is* the teacher's discrete Euler trajectory — a numerical estimate of the integral `∫_t^r v(z_τ, τ; θ_teacher) dτ`, not an exact algebraic identity. They are not the same identity in different notation; they are different objects. The companion O3S algorithm (ternary-search-optimised non-uniform step placement) is a small, portable inference-time trick that should compose with any few-step audio generator. Audio-only paper, mel-spectrogram operating space, **no CPU benchmark, no raw-waveform results, no GFLOPs/sample** — adds a third data point to the audio MeanFlow landscape (alongside SplitMeanFlow on Doubao TTS latent and MeanFlowSE on WavLM+VAE-latent SE) that still leaves raw-waveform CPU-benchmarked MeanFlow as the empty cell.

# What problem does it solve

The vanilla MeanFlow loss `L_MF = ‖u_θ(z_t, r, t) − sg(v(z_t, t) − (t−r)·du_θ/dt)‖²` requires (i) a JVP through `u_θ` to compute `du_θ/dt` and (ii) mixing with an FM target on a fraction of samples ("self-bootstrap with instantaneous-velocity guidance") to keep `u_θ` anchored to the FM solution. The paper claims two pains from this in the speech-synthesis context:

1. **JVP memory + compatibility.** F5-TTS's diffusion transformer requires batch size 1 when trained with vanilla MeanFlow due to JVP memory pressure; `torch.func.jvp` does not compose with flash-attention. Large TTS models become infeasible to train.
2. **Training instability from self-bootstrap.** The "mixing with instantaneous velocity" hyperparameter is delicate, and without strong enough FM guidance the MeanFlow loss collapses (the paper's Table 2 line 3 reports vanilla MeanFlow on CosyVoice2 has decent WER/SIM-o but degraded UTMOS/UV.MOS/CMOS/SMOS — quality is acceptable but speech is unnatural).

The paper's proposal: drop the differential identity entirely. Have a frozen FM teacher numerically integrate `v` over `[t, r]` using `n` Euler steps; supervise the student's average-velocity prediction directly against `displacement / (r − t)`. **The student never differentiates anything; the teacher's discrete trajectory is the target.**

# Method

## (1) Integral velocity distillation (§3.1)

The student parameterises the average velocity `u_student(z_t, t, r)` over interval `[t, r]`. The teacher is a pretrained FM with vector field `v(z_τ, τ; θ_teacher)`.

For each training sample:

1. Sample `t, r` with `0 ≤ t ≤ r ≤ 1` (the paper's convention; the integral is `∫_t^r v dτ` running forward).
2. Sample initial point `z_t`.
3. Discretise `[t, r]` into `n` sub-intervals with steps `t_0 = t, t_1, ..., t_n = r`. Run an Euler integration through the teacher:
   ```
   z_{t_{k+1}} = z_{t_k} + (t_{k+1} − t_k) · v(z_{t_k}, t_k; θ_teacher)     (Eq. 4)
   ```
4. Compute the discrete teacher displacement:
   ```
   Δz_teacher = Σ_{k=0}^{n−1} (t_{k+1} − t_k) · v(z_{t_k}, t_k; θ_teacher)    (Eq. 5)
   ```
   This is the standard Euler-quadrature approximation of `∫_t^r v(z_τ, τ; θ_teacher) dτ`.
5. Normalise by interval length to get the **teacher's average velocity over [t, r]**:
   ```
   v̄_teacher(z_t, t, r) = Δz_teacher / (r − t)                                (Eq. 6)
   ```
6. Distillation loss:
   ```
   L_distill = E_{t,r} ‖u_student(z_t, t, r) − v̄_teacher(z_t, t, r)‖²       (Eq. 8)
   ```

No JVP. No student self-bootstrap. No CFG dropout on the student (the teacher's `v` already bakes in a fixed CFG scale; student is trained CFG-free).

The continuous form being approximated is `v̄(z_t, t, r) = (1/(r−t)) ∫_t^r v(z_τ, τ; θ_teacher) dτ` (Eq. 7). The paper does not claim this is exact — Eq. 5 is a first-order Euler approximation; the teacher NFE `n` is a hyperparameter (sweep in Table 3).

## (2) Initialization strategy (§3.3)

To warm-start the student from an existing FM checkpoint that conditions only on `t`, IntMeanFlow extends the time embedding to take both `t` and `r`:
```
e_t = E(t),  e_r = E(r),  e_{t,r} = [e_t, e_r],  e'_{t,r} = W · e_{t,r}    (Eq. 9)
W = [D_diag, 0]                                                            (Eq. 10)
```
The projection `W` is initialised so that the `r`-side is zero and the `t`-side is identity (`D_diag`). At step 0, the student behaves identically to the teacher; gradient updates fill in the `r`-dependence. **Same trick as iMF / TiM's decoupled `(t, Δt)` embedding** but with explicit identity-init to preserve teacher behaviour at iteration 0.

## (3) Optimal Step Sampling Search (O3S, §3.2)

Companion inference-time trick. The student is trained on continuous `(t, r)` ∈ [0,1]²; at inference, the user picks `n` steps. The paper observes empirically (Fig. 2) that quality as a function of step placement is **near-convex** for each step position given the others fixed.

**Algorithm:** Initialise `T_best = {0, 1/n, 2/n, ..., 1}` (uniform). For each `i ∈ {n−1, ..., 1}` (descending), apply ternary search on `t_i ∈ [t_{i−1}, t_{i+1}]` to maximise a quality metric `L` (paper uses SIM-o on a held-out dev set). Iterate over `i` until no improvement for `p = n` consecutive sweeps. Returns a per-model-specific, **non-uniform** sampling schedule. Empirically improves test-en/test-zh objective and subjective metrics "by a large margin" without increasing NFE (Table 1 lines 6→7 and 16→17).

O3S is structurally **orthogonal to the distillation recipe** — it could be applied to any few-step generator that takes continuous `(t, r)` as input. The natural composition is "O3S applied to a SplitMeanFlow-distilled student" or "O3S on a vanilla pruned-step FM" — untested in the paper.

# How this differs from SplitMeanFlow — the central question

Both come from the same ByteDance authors (Wei Wang, Rong Cao, Yi Guo, Zhengyang Chen, Kuan Chen are on both papers); both target speech audio; both eliminate the JVP; both use `u_θ(z_t, t, r)` parameterisation. **They are not the same trick.** The "integral form" framing applies to both — but the integral is invoked differently.

| Axis | SplitMeanFlow (2507.16884) | **IntMeanFlow (this paper)** |
|---|---|---|
| What gets distilled | Student against itself, via algebraic identity | Student against teacher's numerical integration |
| Identity / target | **Exact** algebraic identity on `(t−r)·u_θ`: `(t−r)·u_θ(z_t,r,t) = (s−r)·u_θ(z_s,r,s) + (t−s)·u_θ(z_t,s,t)` (Eq. 10) | **Approximate** numerical integration through teacher: `v̄_teacher = (1/(r−t)) Σ (t_{k+1}−t_k)·v_teacher(z_{t_k}, t_k)` (Eqs. 5–6) |
| What is `s`? | A split point on `[r, t]` — only the student is queried at `s` | A grid `t_0, ..., t_n` of teacher Euler steps along the trajectory |
| Forward passes per training step | 3 (student-only) | 1 (student) + `n` (teacher); paper sweeps `n ∈ {2, 4, 8, 16}` |
| Teacher role | **Boundary anchor at fraction `p ≥ 0.5` of samples** with `r = t` — student matches teacher's instantaneous `v` at one point | **Full trajectory integration** — teacher does the entire path computation, student matches the resulting interval-average |
| Is it self-supervised? | Yes — Interval Splitting Consistency is identity on the student's own predictions | No — pure teacher distillation; identity reduces to "match teacher" |
| What this is **conceptually** | A finite-interval generalisation of the MeanFlow differential identity, applied to the student's own `u_θ` — proven to recover MeanFlow as the `s → t` limit | **Multi-step trajectory distillation** (Knowledge-Distillation-style) reparameterised through average-velocity output |
| Boundary anchor needed to avoid collapse? | Yes (`p ≥ 0.5` of samples with `r = t`, teacher's `v` supplies target) | No — teacher's full trajectory is the target everywhere; collapse impossible |
| Training cost vs FM teacher | ~3× (3 student forwards) | **~`n+1`× per sample** (teacher does `n` forwards, student does 1) — paper Table 3: time/step = 0.93s (FM) → 1.12s (n=2) → 5.17s (n=16) |
| Can it train from scratch? | Yes (paper notes it converges, doesn't validate) | **No** — the recipe is intrinsically teacher-dependent |

**The structural read.** SplitMeanFlow is a **strict generalisation of MeanFlow** at the loss level — the algebraic identity reduces to MeanFlow's differential identity as `s → t`. IntMeanFlow is a **completely different object**: it is multi-step trajectory distillation through an average-velocity parameterisation. The name "MeanFlow distillation via Integral Velocity" is slightly misleading — the only thing connecting it to the MeanFlow family is the output parameterisation (`u_θ(z_t, t, r)` instead of `v_θ(z_t, t)`). The training loss is structurally closer to **DMD / progressive distillation** than to MeanFlow / SplitMeanFlow.

**Why this matters.** The paper does not cite SplitMeanFlow despite sharing authors. They are positioned as solutions to the same problem (JVP-free, few-step MeanFlow-style speech generation) but operate on completely different design axes. The wiki's MeanFlow-family table now has *two* ByteDance audio entries from the same lab via different mechanisms — which strongly suggests the lab is treating them as parallel deployment options, not successor/predecessor. The IntMeanFlow numbers in this paper (F5-TTS 3-NFE, CosyVoice2 1-NFE) are achieved **without** SplitMeanFlow's algebraic identity; SplitMeanFlow's Doubao deployment was achieved **without** IntMeanFlow's teacher-Euler trajectory. They are two different routes to the same target.

# Architecture and scale

The paper deliberately does *not* introduce a new architecture — the student inherits the teacher's network and adds only the `r`-input via the dual-time-embedding trick (Eq. 9-10).

Two teacher families evaluated:

- **F5-TTS** for text-to-mel: small (158 M, LibriTTS 585 h), base (336 M, Emilia 95k h), medium (592 M, Emilia 95k h) diffusion-transformer.
- **CosyVoice2** for token-to-mel: official pretrained checkpoint (size not disclosed); a flow module converts time-aligned semantic tokens to mel-spectrograms.

**Operating space: mel-spectrogram** in both cases. This is the third audio MeanFlow paper to operate on a frozen intermediate representation rather than raw waveform (alongside SplitMeanFlow on Seed-TTS acoustic latent and MeanFlowSE on WavLM/WaveVAE features). **No raw-waveform results anywhere in the paper.**

# Inference cost — the headline numbers

The paper's RTF benchmarks are NVIDIA A100, batch size 1, RTF measured on the **flow module only** (excluding vocoder/LM):

**Text2Mel (F5-TTS, Table 1):**

| Model | NFE | WER (en) | SIM-o (en) | UTMOS (en) | RTF |
|---|---|---|---|---|---|
| Base FM teacher | 32 | 1.87 | 0.67 | 3.70 | 0.243 |
| Base + IntMeanFlow + O3S | 3 | **1.60** | 0.65 | 3.79 | **0.021** |
| Base + IntMeanFlow + O3S | 2 | 2.04 | 0.63 | 3.24 | 0.013 |
| Base + IntMeanFlow | 1 | 7.27 | 0.48 | 1.84 | 0.009 |

3-NFE matches the 32-NFE FM teacher on objective metrics with **~12× RTF speedup** (0.243 → 0.021). 1-NFE is unusable (WER 7.27 vs human 2.14, UTMOS 1.84 vs human 3.52); **the text2mel task requires 3 NFE to be acceptable**, the paper's headline claim is honest about this.

**Token2Mel (CosyVoice2, Table 2):**

| Model | NFE | WER | SIM-o | UTMOS | RTF |
|---|---|---|---|---|---|
| CosyVoice2 teacher | 32 | 2.17 | 0.66 | 4.36 | 0.510 |
| CosyVoice2 + vanilla MF | 1 | 2.11 | 0.62 | 3.96 | 0.026 |
| CosyVoice2 + IntMeanFlow | 1 | 2.18 | 0.63 | **4.28** | 0.026 |

1-NFE works for token2mel — but **the WER/SIM-o is comparable for vanilla MeanFlow at 1 NFE**, what IntMeanFlow buys here is UTMOS/UV.MOS (4.28 vs 3.96 / 4.47 vs 3.85) — quality, not speed. Vanilla MeanFlow's naturalness collapse is the bug IntMeanFlow fixes. **~20× RTF speedup over the 32-step teacher** (0.510 → 0.026).

**Teacher-NFE / training-cost tradeoff (Table 3):** Time per step scales roughly linearly with teacher NFE — 0.93s (FM baseline, n=0) → 1.12s (n=2) → 1.74s (n=4) → 2.91s (n=8) → 5.17s (n=16). At `n=16`, training is **5.6× slower** than vanilla FM. Lines 8/9/10 of Table 1 sweep `n ∈ {16, 4, 2}` at fixed student size — smaller teacher NFE causes "noticeable performance loss on objective metrics" (WER 1.60 → 1.71 → 1.83 on test-en; line 8/9/10), subjective metrics largely intact. **The cost surface here is "train-time NFE for better student quality"** — opposite axis from inference-NFE.

**No CPU benchmark.** No GFLOPs. No raw-waveform results. Mel-spectrogram + A100 RTF only — the missing axes are the project's headline questions.

# Position in the audio-MeanFlow landscape

Three audio-MeanFlow papers now exist. The cross-paper picture:

| Axis | SplitMeanFlow (2507.16884) | MeanFlowSE (2509.23299) | **IntMeanFlow (this paper)** |
|---|---|---|---|
| Task | TTS acoustic-feature generation | Speech enhancement (noisy → clean) | TTS (text2mel + token2mel) |
| Operating space | Seed-TTS acoustic-diffusion latent | WavLM-Large SSL + WaveVAE latent | **mel-spectrogram** |
| Loss | Algebraic interval-splitting (JVP-free) | Geng's vanilla MeanFlow + JVP | **Teacher-Euler integral distillation (JVP-free)** |
| Self-bootstrap? | No (uses identity, no JVP) | Yes (vanilla MeanFlow) | **No (teacher does the work)** |
| Teacher required? | Yes (recommended two-stage; from-scratch supported but unevaluated) | No (from-scratch on 8×4090) | **Yes — intrinsic to the recipe** |
| Inference NFE | 1–2 | 1 | **1 (token2mel) / 3 (text2mel)** |
| GPU benchmark | Wall-clock vs FM teacher (relative) | RTF 0.013 on 4090 | RTF 0.021–0.026 on A100 (flow module only) |
| CPU benchmark | **No** | **No** | **No** |
| Deployment scale | Doubao TTS (production) | Smallest published audio MeanFlow (40.7 M, 8×4090) | Mid-scale TTS (158–592 M params) |
| Same ByteDance authors as SplitMeanFlow? | — | No | **Yes (5/6 authors overlap)** |

**Three structural reads:**

1. **ByteDance has now published two different JVP-free MeanFlow recipes on speech audio.** SplitMeanFlow is the algebraic-identity recipe (Doubao TTS); IntMeanFlow is the teacher-integral-distillation recipe (CosyVoice2 + F5-TTS). The same authors have published *both*. This is strong evidence that they are not regarded as competitors — they are different recipes for different deployment contexts (probably: SplitMeanFlow when you can afford the from-scratch-compatible identity, IntMeanFlow when you already have a strong FM teacher and want a simpler distillation pipeline).
2. **Mel-spectrogram is now the dominant audio-MeanFlow operating space.** Three of three audio MeanFlow papers operate on a frozen intermediate representation — none on raw waveform. **Raw-waveform unconditional CPU-benchmarked MeanFlow is the empty cell, again.**
3. **Text-to-mel requires 3 NFE; token-to-mel works at 1 NFE.** This is the cleanest published statement of the "1-NFE is task-dependent" caveat in the audio MeanFlow literature. The text2mel student has to do both alignment (text-to-spectrogram time correspondence) *and* spectrogram synthesis in one step; the token2mel student only does spectrogram synthesis since alignment is already in the input tokens. The general lesson for raw audio: 1-NFE is most feasible when the conditioning is dense and time-aligned (mel, tokens, SSL features); sparse conditioning (text, prompts) demands few-step generation.

# How this translates to raw audio

The training loss is dimension-agnostic and immediately portable to a raw-waveform FM teacher. Concretely:

1. **Pretrain a raw-waveform FM teacher** (CRASH-style 1-D conv U-Net, RFWave STFT-domain ConvNeXt, or Flow2GAN's FM pretrain stage).
2. **Add `r`-input to the student** via Eq. 9–10 identity-init projection.
3. **Train** with Eq. 5–8: Euler-integrate the teacher over sampled `(t, r)`, regress student's `u_θ(z_t, t, r)` against `Δz_teacher / (r−t)`.
4. **Run O3S** at inference to optimise step placement under the target NFE.

What's specifically *not* in the paper but needed for our use case:

- **CPU benchmark** — absent. A100 RTF 0.021 → CPU RTF unknown.
- **Raw-waveform sample-space results** — none. Mel-spec only.
- **FAD / MIND / Coverage** — paper uses WER/SIM-o/UTMOS/UV.MOS/CMOS/SMOS (all TTS-quality-oriented), which don't translate to unconditional waveform generation.
- **GFLOPs/sample** — absent.

**What the recipe contributes to the project that the existing papers don't.** It is the **first audio paper in the wiki to demonstrate few-step MeanFlow-style generation on a *general-purpose* TTS model (F5-TTS) rather than a closed proprietary system (Seed-TTS) or a small custom DiT (MeanFlowSE).** F5-TTS code is open; the recipe can be reproduced. The 3-NFE F5-TTS result at near-teacher quality is a relevant data point if we end up with a text-conditioned audio FM in this project.

**What's specifically *worse* about this recipe for our project compared to SplitMeanFlow.** Training cost. At teacher NFE = 16 the training-step time is 5.6× the vanilla FM baseline (Table 3); SplitMeanFlow's three student forwards is ~3× vanilla FM (no teacher in-the-loop at every step). For a from-scratch raw-waveform pipeline where we have to train *both* the FM teacher and the few-step student, the IntMeanFlow recipe roughly doubles the total compute compared to SplitMeanFlow.

# Strengths

- **No JVP, no self-bootstrap, no curriculum, no algebraic identity** — the simplest possible MeanFlow-style training recipe at the loss level. Anyone with a pretrained FM teacher can implement this in ~20 lines.
- **Identity-init dual-time embedding (Eq. 9–10)** is a small, robust trick that ensures the student starts at zero distillation gap. Portable to any fastforward recipe that adds time-pair conditioning.
- **O3S is a clean, plug-in inference-time improvement** with ternary-search optimisation of step placement. Composes with any few-step generator that takes continuous `(t, r)` input. **Plausibly the most reusable contribution of the paper.**
- **Demonstrates the FM-teacher-NFE / student-quality tradeoff explicitly** (Table 3). Tells you what training-cost budget buys you what student quality.
- **Token2mel at 1-NFE matches teacher quality on objective metrics** while improving UTMOS over vanilla MeanFlow — first clean published comparison of IntMeanFlow vs vanilla MeanFlow at matched architecture (CosyVoice2).
- **Trained at industrial scale on Emilia (95k hours)** — comparable to the largest open audio FM training runs.

# Limitations and open questions

- **No CPU benchmark, no GFLOPs, no raw-waveform results.** Project-headline metrics are all absent.
- **No comparison to SplitMeanFlow** despite the author overlap. The natural A/B (FM teacher → SplitMeanFlow vs FM teacher → IntMeanFlow at matched compute) is missing.
- **Training cost scales linearly with teacher NFE.** At `n = 16` training is 5.6× FM-baseline cost; the recipe is not free. If teacher is a very large model, training time dominates.
- **Intrinsically teacher-dependent.** Unlike SplitMeanFlow (algebraic identity stands alone), IntMeanFlow has no from-scratch path. The recipe is post-training only.
- **Eq. 5–6 is a first-order Euler approximation.** Higher-order quadrature (Heun, RK4) would reduce target bias for fixed `n` — paper does not explore. The numerical error of the integral approximation directly biases the regression target; this is an obvious axis for refinement.
- **No sub-interval sampling distribution sweep.** The paper picks how `t_0, ..., t_n` is constructed within `[t, r]` but does not ablate uniform vs log-spaced vs adaptive sub-grids on teacher NFE.
- **Same `n` for all `(t, r)` pairs.** Small `(r − t)` intervals are well-approximated by few Euler steps; large `(r − t)` intervals need more. Adaptive teacher NFE per training example is unexplored.
- **No `n=1` baseline** — what does "teacher does one step" reduce to? Probably equivalent to single-step DMD-style distillation. Would clarify the relationship to existing trajectory-distillation literature.
- **CFG handling at distillation time is fixed.** Teacher uses CFG scale 3.0 in F5-TTS experiments; CFG is baked into student. The iMF-style flexible-CFG-as-conditioning extension is uncited and untried.
- **O3S uses SIM-o as the optimisation metric** — but any quality metric would work. For unconditional audio, what's the right `L`? FAD on a dev set is the natural choice but the search is expensive (each metric eval requires generating an entire dev set).
- **MeanAudio (2508.06098, cited as [21])** is the closest prior work — a text-to-audio MeanFlow paper — but is not directly compared. The wiki should ingest it eventually.

# Things to test in this repo

1. **Port to a CRASH-style 1-D conv U-Net for unconditional drum generation.** Pretrain CRASH-FM teacher; distill with IntMeanFlow at `n ∈ {2, 4, 8}` teacher NFE; report 1-NFE / 3-NFE FAD, MIND, CPU xRT. The cleanest first audio-domain port.
2. **Direct A/B IntMeanFlow vs SplitMeanFlow on the same backbone.** Same teacher, same data, same compute. The natural experiment the paper itself skips. **Single most informative experiment for the audio MeanFlow design space.**
3. **Add raw-waveform CPU benchmark.** Mel-spec RTF 0.021 on A100 → ?? on Xeon at 1 s @ 22 kHz. Without this, the project headline cannot place this paper.
4. **O3S on SplitMeanFlow-distilled student.** O3S is orthogonal to distillation recipe; should improve any continuous-`(t, r)` few-step generator. Compose with the Doubao recipe.
5. **Higher-order quadrature for the teacher integration.** Replace Euler with Heun (2 evals per sub-step) or RK4 (4 evals per sub-step) — does the reduced target bias translate to better student quality at matched total teacher cost?
6. **Adaptive teacher NFE.** Use small `n` for small `(r − t)`, large `n` for large `(r − t)` — same total teacher cost amortised better. Or: schedule `n` over training, larger early (informative target) → smaller late (cost saving).
7. **Distillation loss weighting by interval length.** Currently uniform; small `(r − t)` intervals have small target magnitude and so contribute less to the loss naturally — but the paper doesn't validate this. Sweep `‖·‖² / (r−t)^α` for `α ∈ {0, 1, 2}`.
8. **Test what fails at 1-NFE for text2mel.** The 1-NFE text2mel collapse (WER 7.27, UTMOS 1.84) is a strong negative result — what specifically fails? Time alignment? Spectral envelope? Voice identity? Diagnose; this may tell us about general 1-NFE limits on unaligned conditioning.
9. **Flexible-CFG extension (iMF-style).** Bake `ω` as a conditioning input rather than fixed CFG scale at distillation. Should compose with the recipe at zero quality cost.
